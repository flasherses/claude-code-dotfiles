# /audit — 安全审计

对指定文件或项目执行全面安全审计。

## 用法

```
/audit [文件或目录路径]
```

不指定路径时，审计整个当前工作目录。

## 执行流程

1. **确定审计范围**
   - 如果提供了 `$ARGUMENTS`，审计指定文件
   - 否则审计当前目录下所有源代码文件

2. **安全审计** — 调起 `security-auditor` 子代理
   - 按 OWASP Top-10 逐项扫描
   - 硬编码密钥检测
   - 注入漏洞扫描（SQL/XSS/命令）
   - 认证与授权检查

3. **输出报告** — 按严重等级分组（Critical / High / Medium / Info）

## 输出

- 审计文件数
- 按 CWE 编号分组的问题清单
- 每个问题的攻击场景和修复建议
- 整体风险评估：LOW / MEDIUM / HIGH / CRITICAL
